Section 2: MCP Server — Tools Claude Can Call

These are the tools your MCP server exposes. Claude calls them like functions.

CORE

deploy

Deploys code to a new or existing Vercel project. The "publish this" magic.

deploy({
  project_name: "q4-sales-dashboard",
  files: { "index.html": "...", "app.js": "...", ... },
  framework: "nextjs" | "static" | "vite",
  env_vars: ["OPENAI_API_KEY", "SUPABASE_URL"],  // references to vault
  description: "Sales dashboard with AI insights",
  folder: "/marketing/dashboards"
})
→ Returns: { url: "q4-sales-dashboard.acme.vercel.app", status: "live" }
CORE

get_available_secrets

Lists which API keys/secrets are available for this tenant. Claude sees names, never values.

get_available_secrets()
→ Returns: [
    { name: "OPENAI_API_KEY", description: "GPT-4 access", added_by: "admin" },
    { name: "STRIPE_SECRET_KEY", description: "Payment processing", added_by: "admin" },
    { name: "SUPABASE_URL", description: "Shared database", added_by: "system" },
    { name: "SUPABASE_ANON_KEY", description: "Shared database", added_by: "system" }
  ]
CORE

provision_database

Creates a scoped database schema for a project. Uses the shared Supabase instance with RLS isolation.

provision_database({
  project_name: "q4-sales-dashboard",
  tables: [
    { name: "leads", columns: { name: "text", score: "int", ... } }
  ]
})
→ Returns: { schema: "proj_q4_sales_dashboard", connection_string: "injected as env var" }
MGMT

list_projects

Claude can browse existing company projects — useful for "update the sales dashboard" or "what apps do we have?"

list_projects({ folder: "/marketing" })
→ Returns: [
    { name: "q4-sales-dashboard", url: "...", last_deployed: "2h ago", author: "Sarah" },
    { name: "competitor-tracker", url: "...", last_deployed: "3d ago", author: "Mike" }
  ]
MGMT

get_project

Gets full details + source code of an existing project so Claude can modify it.

get_project({ name: "q4-sales-dashboard" })
→ Returns: { files: {...}, env_vars: ["OPENAI_API_KEY"],
             db_schema: {...}, deploy_history: [...] }
AGENT

schedule_agent

Deploys a background agent that runs on a schedule or responds to webhooks.

schedule_agent({
  name: "daily-report-bot",
  code: "...",
  schedule: "0 9 * * MON-FRI",  // cron: 9am weekdays
  env_vars: ["OPENAI_API_KEY", "SLACK_WEBHOOK"],
  description: "Generates daily sales summary and posts to Slack"
})
→ Returns: { agent_id: "...", next_run: "tomorrow 9:00 AM", logs_url: "..." }
MGMT

get_usage

Claude can answer "how much are we spending?" or "which app has the most traffic?"

get_usage({ period: "last_30_days" })
→ Returns: { total_deploys: 47, api_calls: 12340,
             top_projects: [...], estimated_cost: "$23.50" }

How Claude Discovers These Tools

Claude Code users: Add MCP server to ~/.claude.json or project .mcp.json. Server runs locally, calls your platform API.

Claude.ai users: Remote MCP server via OAuth. Employee authenticates once → Claude can call all tools for their tenant.

Both methods hit the same platform API — the MCP server is just a thin adapter.
