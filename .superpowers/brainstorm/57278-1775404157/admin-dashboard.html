Section 5: Admin Dashboard

Two views: everyone sees the project gallery. Admins see governance & usage.

Page Structure

ALL EMPLOYEES

Project Gallery

Browse all company projects. Search, filter by folder/author/status. Click to open live URL.

My Projects

Projects I created. Quick access to URLs, deploy history, logs.

Folders

Nested folder tree. Teams can organize by department, project type, etc.

ADMIN ONLY

Usage & Analytics

Deploys, API calls, DB usage, costs — per project, per user, over time. Charts + tables.

Secrets Vault

Add/revoke API keys. See which projects use which secrets. Rotate keys.

User Management

Invite/remove employees. Set roles (admin/member). View per-user activity.

Agent Monitor

All running agents. Status, last run, error rate, logs. Pause/resume/kill.

RTL / Hebrew Support

Dashboard (your product)

  • Built with next-intl for i18n
  • Language toggle: EN | HE in header
  • dir="rtl" on <html> when Hebrew
  • Tailwind CSS rtl: variant for layout flips
  • All labels, dates, numbers localized

Published apps (employee-built)

  • CLAUDE.md includes RTL instruction: "When user writes in Hebrew, generate RTL-compatible code with dir=rtl"
  • Auth login page supports Hebrew
  • Not enforced — employees choose per-project
