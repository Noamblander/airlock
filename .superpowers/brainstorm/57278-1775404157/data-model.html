Row-Level Security (RLS)

-- Every table has this policy:
CREATE POLICY tenant_isolation ON projects
  USING (tenant_id = auth.jwt() ->> 'tenant_id');

-- Users only see their company's data.
-- No cross-tenant leaks possible.

Published App DB Isolation

-- Each project gets its own schema:
CREATE SCHEMA proj_q4_sales_dashboard;

-- Tables live inside the schema:
proj_q4_sales_dashboard.leads
proj_q4_sales_dashboard.reports

-- Apps can only access their own schema.
-- Connection string is scoped.

Secrets Encryption

API keys encrypted at rest with AES-256. Decryption key stored in environment variable on the platform server — never in the database. Secrets are decrypted only when injecting into Vercel env vars during deploy.

Usage Events

Append-only log captures every action: deploys, API calls, DB queries, agent runs. Each event has a cost_cents field. This powers the admin dashboard analytics and future billing.
